- **多模块微服务架构**：规划了 `backend-java` (Spring Boot)、`ai-service` (FastAPI) 和 `frontend` (Vue/React) 三个核心服务，通过 REST API 通信。
- **严格职责隔离**：Java 后端负责任务编排与持久化，Python 服务负责 GitHub 数据获取、Semgrep 静态分析及 LLM 推理，前端仅消费 Java 后端接口。
- **中心化文档驱动**：以 `docs/` 目录下的 PRD、架构、API 和数据库设计文档为唯一事实来源，强制执行“文档先行”的开发流程。
- **多智能体协作流水线**：在 `docs/AGENT_RULES.md` 中定义了 Cursor（编码）、Codex（验证）和 Qoder（审查）的角色边界与基于 ChatGPT 架构师的中转交接机制。
- **基础设施占位**：通过 `docker-compose.yml` 预留服务容器定义，使用 `.env.example` 管理敏感配置（如 GitHub Token、LLM Key）。
